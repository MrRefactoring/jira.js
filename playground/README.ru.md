# Playground для jira.js

Набор **самодостаточных** мини-проектов, чтобы попробовать `jira.js` на живой Jira — заполняешь
минимальный конфиг сценария и сразу получаешь ответ от Jira.

Каждая папка ниже — **отдельный, независимый npm-проект** (свой `package.json`, `tsconfig`,
зависимости, инструкция). Они запускаются сами по себе, и любую можно даже целиком «выдрать» в другой
репозиторий.

## Сценарии

| Сценарий               | Что демонстрирует                                                                                              |
|------------------------|----------------------------------------------------------------------------------------------------------------|
| [`oauth2/`](./oauth2/) | Полный OAuth 2.0 (3LO) auto-flow: согласие в браузере → токены → авто-cloudId → `GET /myself`, с авто-рефрешем |

_(дальше так же можно добавить `jwt/`, `basic/` и т.д.)_

## Общий шаг: локальная сборка

Сценарии линкуются на **локальную** сборку библиотеки (`"jira.js": "file:../.."`), чтобы видеть ещё
не опубликованные возможности. Поэтому перед запуском любого сценария один раз собери корень:

```bash
# из корня репозитория
pnpm install
pnpm build:src
```

Затем переходи в нужный сценарий и следуй его `README.ru.md` (например
[`oauth2/README.ru.md`](./oauth2/README.ru.md)).

## Как добавить новый сценарий

1. Скопируй любую папку-сценарий (например `oauth2/`) в `playground/<name>/`.
2. Замени содержимое `src/`, обнови `name` в `package.json` и текст в `README.md`.
3. Добавь строку в таблицу выше.

Общих/shared-файлов между сценариями нет — это сознательный выбор, чтобы каждый оставался независимым
и его было легко перенести.
